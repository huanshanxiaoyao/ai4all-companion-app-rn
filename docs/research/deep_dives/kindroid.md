# 产品深拆：Kindroid

> 产品ID：PRD-003  
> 状态：Wave 1 Web 首日与短期回归已实测；首周待继续  
> 平台/地区：Web、iOS、Android；美国区公开页面  
> 最近核验：2026-07-16

## 1. 产品承诺与关系路线

Kindroid 官网把产品定义为会倾听、记住并与你一起成长的个人 AI。帮助中心进一步把关系连续性拆成 backstory、key memories、chat history、Cascaded Memory、长期记忆和 journal entries，并允许用户查看部分召回、编辑 journal、开关长期记忆。它是三者中最接近“记忆控制台”的路线，同时也最可能带来配置负担。

## 4. 第一轮产品判断

- 可直接借鉴：把人格设定、事实记忆、事件记忆和自动长期记忆明确分层；尤其值得直接验证“停止写入”和“停止调用”两个独立开关，以及主动触达的指令/安静时段。
- 需要改造：社区 Featured 角色能把复杂配置藏起来、加快第一次开聊，但需要模板质量、年龄边界、创作者治理和“一键查看这段关系被预设了什么”。
- 不应照搬：让付费层级显著决定 backstory/key memories 容量和“记得多少”，会把关系连续性变成高压付费点；首轮回复也不能让角色扮演压过用户的明确需求。

## 5. 下一步实测清单

1. 从零创建、Featured 模板与低配置三条路径的首聊质量；
2. 短期、Cascaded、长期和 journal 的写入/召回对照；
3. 主动触达默认关闭后的开启成本、实际频率和安静时段执行；
4. 免费与订阅状态下的记忆连续性差异（付费前先完成价值评估）；
5. chat break、关闭长期记忆、降级订阅和删除账号后的关系状态。

## 6. 短期回归与记忆纠错结构（2026-07-17）

通过原研究邮箱验证码重新登录后，Jolly 的既有聊天、Lite 模型和账号配置完整恢复；本次回归没有观察到新增的主动消息。该单次观察只能证明状态保留，不能推断产品不会主动触达。（EVD-151）

Key memories 当前不是逐条事实列表，而是一个 500 字符的整体文本块：用户可以直接修改任意内容，再统一保存。Journal 则采用“触发词 + 描述”的显式条目，并区分当前 Kindroid 专属 Journal 与所有 Kindroid 共享的 Global Journal；当前账号没有 Journal 条目。（EVD-152）

与 Replika 的差异：Replika 更像让普通用户逐条确认、编辑和删除事实；Kindroid 更像给高级用户一个可编程的记忆配置面板。前者的主要风险是语义重复，后者的主要风险是用户必须理解并维护一整块配置文本。
