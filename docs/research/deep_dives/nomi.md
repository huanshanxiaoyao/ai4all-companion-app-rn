# 产品深拆：Nomi

> 产品ID：PRD-002  
> 状态：Wave 1 公开证据完成；Web 登录入口已实测，账号内流程被 OAuth 阻塞  
> 平台/地区：Web、iOS、Android；美国区公开页面  
> 最近核验：2026-07-16

## 1. 产品承诺与关系路线

Nomi 官网把产品定义为“有记忆和灵魂的 AI 伴侣”，关系可以是有意义的友情、亲密关系或导师。官方还把短期、中期和长期记忆、custom backstory、自然成长、语音和情境自拍放在同一套关系叙事中。其路线不是先锁定单一人设，而是让用户在“明确设定”和“自然相处”之间选择。

Web 实测补充：Nomi 登录页只提供 Google 与 Apple OAuth，没有普通邮箱注册。Google 在 Playwright 自动化浏览器中明确拒绝登录，因此当前没有把公开定位误写成账号内实测；需要用普通 Chrome 人工完成一次 OAuth 后继续。

## 3. 第一轮产品判断

- 可直接借鉴：允许用户选择“先设定”或“先相处”，降低 onboarding 的单一路径假设。
- 需要改造：自然成长必须配套可见、可纠错的关系与记忆控制，否则一致性难以验证。
- 不应照搬：用“灵魂”或高度拟人措辞弱化 AI 身份边界。

## 4. 下一步实测清单

1. 通过普通 Chrome 完成一次 Google OAuth，再采集创建 Nomi 时的身份、关系、外观、backstory 与默认值；
2. 不写 backstory 与详细设定两条路径的首聊差异；
3. 前 20 轮的偏好记忆、跨会话召回和错误纠正；
4. 语音、自拍、主动消息和群聊的出现时机；
5. 免费额度、订阅墙、取消和删除路径。
