# 用户评论抽样与编码规范

## 1. 研究目的

评论用于发现问题、关系价值和流失原因，不直接代表全体用户。任何比例结论必须说明平台、地区、时间窗口、筛选方法、样本量和去重方法。

## 2. 最小记录字段

- 产品、平台、地区、评分；
- 评论日期与应用版本（平台提供时）；
- 评论原文、来源 URL、采集日期；
- 采集方式与抽样规则；
- 主题标签、情绪、是否脱敏、是否重复；
- 编码人和复核状态。

## 3. 一级主题标签

| 标签 | 定义 | 示例判断 |
| --- | --- | --- |
| 关系价值 | 被理解、陪伴感、依恋、共同成长 | 用户是否感觉关系持续存在 |
| 记忆准确 | 正确记住、忘记、错记、无法删除 | 错误记忆单独标记为负向 |
| 对话质量 | 自然度、重复、共情、逻辑、人格一致性 | 区分模型能力与产品交互问题 |
| 主动触达 | Push、回访、频率、时机、打扰 | 同时记录能否关闭 |
| 语音体验 | 音色、延迟、打断、情绪表达、稳定性 | 区分语音消息与实时通话 |
| 视觉/具身 | Avatar、动画、宠物/硬件触感、在场感 | 判断是否真正增强关系 |
| 易用性 | 注册、导航、速度、无障碍和跨设备 | 记录平台与版本 |
| 价格/付费墙 | 价格、试用、退款、权益变化 | 记录地区和币种 |
| 安全/边界 | 依赖、排他、危机回应、未成年人、审核 | 高风险内容单独复核 |
| 隐私/数据 | 录音、监听、记忆、导出、删除账号 | 不推断未公开的数据处理 |
| 稳定性 | 崩溃、登录、网络、更新退化 | 尽量关联版本 |
| 流失原因 | 明确卸载、退款、停止使用或替代产品 | 记录直接原因而非猜测 |

允许多标签。情绪统一使用 `正向 / 中性 / 负向 / 混合`。

## 4. 抽样建议

1. 先按产品、平台、地区和时间窗口分层。
2. 同时覆盖高分、低分和最近版本评论，避免只看极端评价。
3. 第一轮每产品建议 50–100 条可追溯评论；完成主题饱和度检查后再决定扩量。
4. 至少 10% 样本由第二位编码人复核；分歧记录规则修订，不静默覆盖。
5. 同文案、机器人评论、跨平台复制内容进入去重组，不重复计数。

## 5. 不允许的结论

- 未证明抽样代表性时，不说“多数用户”“用户普遍认为”。
- 不把厂商案例当成用户评论。
- 不把评论评分直接等同留存、付费或心理健康效果。
- 不对评论者身份、健康状态和动机做超出文本的推断。
