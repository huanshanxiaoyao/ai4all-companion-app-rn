### 推荐实现

候选 A 的长期部分是 `practice`，当前第一章可直接注册为 `collection`：

| 后端字段 | 草案值 |
| --- | --- |
| slug | `twelve_life_textures` |
| display_name | `生活手感` |
| statement | `和用户一起形成 12 种能让生活重新变得可感的个人方法` |
| target_count | `12` |
| bar | `中：必须是用户亲自经历并确认有帮助或愿意再试的具体方法；不以完成任务为准` |
| short_label | `生活手感` |
| inquiry | `普通生活是否必须被完成、被记录、被证明，才算有意义` |

阶段回顾建议为 3/6/9/12。当前后端可直接完成计数和封顶；第一版清单可在第 12 项时读取全部条目生成，不必先建新表。另需小改完成态渲染：达到 12 后显示“第一版生活手感已形成”，不再提示继续寻找。没有新增记录不影响关系，也不提醒用户“还差几项”。

`mission_moment.content` 建议使用紧凑文本：`方法｜适用情境｜所需力气低/中/高｜用户确认的作用`。不得填入体重、诊断、用药或精确健康指标。

### 候选 B 的实现前置

小院不应是种田游戏：没有每日浇水、枯死惩罚、稀有植物和付费加速。它是关系作品，可由服务端根据少量已确认条目生成阶段快照；访客是否可见要由主人单独决定。

### 候选 C 的安全门槛

实验只限低风险生活体验，如换一个工作位置、尝试一道简单食物、散步观察一种颜色。涉及饮食限制、运动强度、药物、睡眠剥夺、花钱或联系第三人时不得由角色自行发起。
